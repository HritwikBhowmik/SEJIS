The backend service is written which is located under app folder and running on 0.0.0.0:6969. The documentation is here:  http://0.0.0.0:6969/docs. You have to design and develop the front-end or client side application only and only using basic HTML, CSS and javascript, and remember DO NOT  USE ANY frontend frameworks like react, next or nuxt please. 

First there will be a attractive landing page representing SEJIS(software engineering job interview simulator), from landing page user can go to registration and login page. The registration and login section will held on same page, like when user click on registration->registration form will appear, when click on sign-in -> sign in interface will appear. For authentication the APIs are implementated, for registration -> /api/v1/auth/register and for login -> /api/v1/auth/register. 
After a successfull user login, a fine and modern dashboard will appear. A table(please read the "./app/models/model.py -> class InterviewScore") will be there which fetch data using -> /api/v1/dashboard/dashboard. And a button, by clicking that button the user will redirected to a new interface where the interview will begin.
First in that interface, a pdf upload section will be there where user/candidate can upload the pdf -> /api/v1/resume/cv-upload. The API will return the job_role, print the response and  also say out loud using javascript's web speech API. After that there will be "Start Interview" buttion appear, by clicking that button the user can start participating in the interview. The button will trigger -> /api/v1/interview/intro-ques which will returns a introductory question for the candidate, in frontend the question will print and also say out loud  using web speech api. After that a 5 minute timer will be started, the timer will visible to the user. The frontend will take the user response by voice/speech and converted the speech into text using web speech api of javascript. By clicking on a button called "Next Phase" button a the answer of the candidate will evaluated by -> /api/v1/eval/intro-eval (this api's response will not visible to the user) and after 5 minute of timer end the api will call automatically.
After that a new phase will start. This API -> /api/v1/interview/dsa-ques will return a DSA related question, in frontend the question will printed and a voice will be say out loud using web speech api of javascript (The voice is "Solve this and you'll get this job!"). A code editor like text field will be there, where the candidate will write the code. And a 15 minute of timer will start and visible to the user. When clicked "submit code" button or the timer is out of time, the user's response will evaluated by -> /api/v1/eval/dsa-eval (response will not visible to the user) and next phase  will start.
Here a system design question will be asked to the candidate, question will come  from /api/v1/interview/sysd-ques. The question will be printed in the frontend and say out load by the web speech api of javascript. As response user will submit a image containing system design in -> /api/v1/eval/sysd-eval. The evaluation result will not be shown to the user. Like previous phases a timer (15 minute) will be there. After completing this phase the final phase will started, the question will get from here /api/v1/interview/final-ques  . Print the question in front end and say out loud using web speech api of javascript. 5 minute of timer will be started. A user's speech response will be taken and converted to text using web speech api and send to -> /api/v1/eval/final-eval for evaluation (response will not visible to the user). A button will appear "submit interview", by clicking that a API will call -> /api/v1/interview/submit-interview (here it'll save all the mark and ai generated report). Also another button will appear named "Download Report" it'll trigger /api/v1/download/interview/{record_id}/pdf   , this will generate and download the pdf report to the user. Add a download report button on each row of the dashboard's table so that user can download particular pdf report of particular interview record id.
